Project Architecture

A high-performance spatial data router designed to serve and extract authoritative Dutch geographical boundaries. The architecture operates as a static GeoJSON Delivery API, natively routing, rendering, and exporting complex Polygon and MultiPolygon geometries to the client.

  • Hierarchical Boundary Modes: Dynamic data layer switching allowing users to drill down from macro Province (Provincie) borders, into specific Municipalities (Gemeente), and down to localized Neighborhoods (Buurt).
  • Personalized Extraction Pipeline: Users can isolate custom combinations of political boundaries and download exact, coordinate-based GeoJSON polygons representing their personalized selection.
  • State Management: Client-side logic safely serializes raw multi-axis coordinates through the DOM, preserving geometry fidelity across user sessions and selections.

Technical Implementation

The backend manages a repository of static GeoJSON files, acting as an optimized file router. It routes HTTP requests based on the selected administrative hierarchy (Province, Gemeente, or Buurt), transmitting only the necessary coordinate arrays to the Leaflet.js renderer. This minimizes memory overhead and latency. During extraction, the Vite-powered frontend parses the stored geometry of the user's custom selection and dynamically assembles a pristine `.geojson` file for immediate local download.

Interface & Spatial Rendering

The map interface is engineered to cleanly parse high-density polygon data, breaking down complex nationwide coordinates into an easily readable and selectable spread.
